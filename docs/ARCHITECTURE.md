# Architecture & Features

A reference for how Super Nonogram is put together today: what it does,
what's in each folder, and how the code paths fit together. Intended for
someone returning to the codebase after a break, or someone new who
wants to land a change without spelunking blindly.

## What the app is

A Flutter implementation of the classic nonogram (a.k.a. picross) puzzle
game. The twist is that puzzles aren't hand-authored — they're generated
on demand, either from an image search (Pixabay) or procedurally from a
level number / seed.

- **Framework:** Flutter (Dart `>=3.10.0 <4.0.0`, Flutter `>=3.41.0`)
- **App version:** `0.12.0+12000` (`pubspec.yaml`)
- **iOS bundle id:** `com.adilhanney.supernonogram`
- **Distribution:** App Store, Play Store, web (PWA), Windows installer,
  macOS, Linux (Flathub + AppImage)

## Supported platforms

iOS, Android, Web (PWA), macOS, Windows, Linux. Each has its own
top-level folder (`ios/`, `android/`, `web/`, `macos/`, `windows/`,
`linux/`) generated by Flutter and lightly customised.

## Navigation flow

Routes are declared in `lib/main.dart:15-44` with `go_router`:

| Path | Screen | Notes |
|---|---|---|
| `/` | `TitlePage` (`lib/pages/title_page.dart`) | Home menu |
| `/search` | `SearchPage` (`lib/pages/search_page.dart`) | Pixabay search |
| `/play?query=<encoded>` | `PlayPage(ImageGameMode)` | Puzzle from image search |
| `/play?level=<int>` | `PlayPage(LevelGameMode)` | Procedural level |
| `/play?classic=<seed>` | `PlayPage(ClassicGameMode)` | Seeded random board |
| `/settings` | `SettingsPage` | Font & haptic toggles |

`main()` (in `lib/main.dart`) waits for the persisted `currentLevel` to
load, registers the Righteous font license, signs into Game Center / Play
Games if supported, then mounts `MyApp` which holds the `MaterialApp.router`.

## Game modes

`lib/data/game_mode.dart` defines a sealed-style union:

- `ImageGameMode(String query)` — puzzle generated from Pixabay results
  for the query.
- `LevelGameMode(int level)` — deterministic procedural level. Board
  size and fill probability scale with the level number
  (`lib/api/level_to_board.dart`).
- `ClassicGameMode(int? seed)` — seeded random nonogram with no image
  backing (`lib/api/classic_puzzles.dart`).

Each mode knows how to produce the board the `PlayPage` will render.

## Board pipeline

The board is the heart of the app and lives in `lib/board/`.

1. **State** — every cell is a `TileState` (`empty | selected | crossed`)
   in `lib/board/tile_state.dart`. The board itself is a 2D list of
   `ValueNotifier<TileState>`s held by `lib/board/board.dart`.
2. **Clues** — `lib/board/board_labels.dart` walks each row and column,
   collapsing runs of `selected` cells into the list of run-lengths that
   nonograms display along the edges.
3. **Rendering** — `board_grid.dart` arranges the grid plus its row/col
   labels; the heavy painting goes through custom RenderObjects in
   `lib/board/render_objects/` for tile and label perf.
4. **Input** — `board.dart` handles taps, right-clicks / stylus-button
   presses (for `crossed`), and pan gestures that drag-fill a line.
5. **Win check** — the `isSolved` getter in `board.dart` compares the
   current run-length labels against the answer's labels and triggers
   the achievement callback when they match.
6. **Toolbar** — `lib/board/toolbar.dart` lets the user switch which
   state taps apply.
7. **Serialisation** — `lib/board/ngb.dart` implements the `.ngb`
   on-disk format used for cached puzzles.

## Puzzle generation

- **Pixabay** — `lib/api/api.dart` queries the Pixabay image search API
  and returns image URLs.
- **Image → board** — `lib/api/image_to_board.dart` downloads an image,
  downsizes it to puzzle-grid resolution, thresholds pixels into
  `selected`/`empty`, and wraps the result as a `NonogramBoard`.
- **Procedural levels** — `lib/api/level_to_board.dart` maps a level
  index to a size and fill probability (`pValue`), so early levels are
  small and sparse and later levels are larger and denser.
- **Classic seeded** — `lib/api/classic_puzzles.dart` uses a seeded RNG
  so the same seed reproduces the same board.

## Persistence

- `lib/api/file_manager.dart` is the storage abstraction. On native
  platforms it uses the real filesystem (`path_provider`); on web it
  falls back to `SharedPreferences` because there is no fs.
- `lib/data/stows.dart` exposes app-wide settings as observable
  "stows" (from the `stow_plain` package): things like `currentLevel`
  and the `hyperlegibleFont` toggle. Widgets `addListener` to a stow
  and rebuild when it changes.
- Cached image puzzles persist as `.ngb` (board) + `.json` (metadata)
  + `.png` (source image) triples.

## Theme & accessibility

`lib/theme/theme.dart` builds Material 3 themes for both brightnesses,
respecting `MediaQuery.highContrastOf` and tweaking per `defaultTargetPlatform`.
Accessibility features:

- Atkinson **Hyperlegible** font toggle (`stows.hyperlegibleFont`).
- High-contrast color schemes when the OS asks for them.
- Haptic feedback via `lib/util/sonic_controller.dart` (audio is still
  a TODO in that file).

## Achievements

`lib/games_services/games_services_helper.dart` wraps the
`games_services` package; `achievement_ids.dart` holds the Android and
iOS achievement identifiers for levels 1, 10, 50, 100, 500, 1000.
Sign-in is attempted in `main()` only on supported platforms.

## Localization

Powered by `slang` (config in `slang.yaml`). Source strings live in
`lib/i18n/`. Currently shipping English and Italian. Adding a new
language is the contribution path documented in the existing
`README.md`.

## Testing

`test/` covers the parts of the app that benefit most from unit-level
checks:

- `board_labels_test.dart` — clue-derivation logic
- `board_golden_test.dart` / `golden_screenshot_test.dart` — visual
  regression
- `image_to_board_test.dart` — image conversion
- `level_to_board_test.dart` — procedural generator determinism
- `pixabay_api_test.dart` — Pixabay client with a mocked channel
- `a11y_tile_test.dart` — accessibility semantics on tiles
- `util/mock_channel_handlers.dart` — shared test helpers

Run with `flutter test`. CI runs the same in `.github/workflows/test.yaml`.

## CI / build

`.github/workflows/` has one file per release surface — `ios.yml`,
`android.yaml`, `macos.yml`, `linux.yml`, `windows.yml`, `web.yaml` —
plus `test.yaml` for the Dart unit tests on every push.

## Features at a glance

- Generate puzzles from a Pixabay image search.
- Play numbered procedural levels with auto-scaling difficulty.
- Play seeded classic random boards.
- Three tile states: empty, filled, crossed-out.
- Pan / drag to fill a row or column quickly.
- Tap-style toolbar to switch what tap-fill applies.
- Automatic win detection against the answer's clues.
- Game Center / Play Games achievements at milestone levels.
- Material 3 dynamic theming, light/dark, high-contrast aware.
- Atkinson Hyperlegible font toggle for readability.
- Optional haptic feedback.
- Cached puzzles persist offline (`.ngb` / JSON / PNG triples).
- English & Italian UI; community-extensible.
- Ships on iOS, Android, Web, macOS, Windows, Linux.
